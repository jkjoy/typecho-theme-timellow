# Timellow

`Timellow` 是一个为 Typecho 制作的极简主题，首页、归档页、文章页、评论区和独立页面保持统一风格，并针对友情链接、归档、分类、标签等页面做了专门适配。

## 安装

1. 将主题目录放到 `usr/themes/typecho-theme-timellow`
2. 登录 Typecho 后台
3. 进入 `控制台 -> 外观`
4. 启用 `Timellow`

## 主题设置

在 `控制台 -> 外观 -> Timellow -> 设置外观` 中可以配置：

- `Favicon 地址`
- `站点副标题`
- `备案号`
- `页脚补充文案`
- `默认缩略图`
- `统计代码`
- `文章列表加载方式`
  - `页码`
  - `加载更多`
- `全站字体`
  - `霞鹜文楷（内置）`
  - `自定义字体`
- `自定义全站字体名称`
- `自定义全站字体文件 URL`

## 全站字体

主题默认内置 `assets/fonts/lxgw.woff2`，启用后会全站生效。

如果你要换成其他字体：

1. 在主题设置中将 `全站字体` 切换为 `自定义字体`
2. 填写 `自定义全站字体名称`
3. 填写 `自定义全站字体文件 URL`

支持的字体格式：

- `woff2`
- `woff`
- `ttf`
- `otf`

## 友情链接页面

主题内置了友情链接独立页模板。推荐新建一个页面，`slug` 使用：

```text
links
```

主题会按下面的顺序读取友情链接数据：

1. 优先读取友链插件数据表 `table.links`
2. 如果没有使用插件，或插件中没有可用数据，则回退到解析友情链接页面内容

### 页面内容解析格式

推荐把友链数据写在页面内容的注释块中，这样页面说明文字可以正常显示，友链数据本身不会直接渲染到页面上。

示例：

```html
这里可以写友情链接页面的说明文字。

<!-- timellow-links-start -->
# 设计
Typecho | https://typecho.org/ | 轻量开源博客程序 | https://typecho.org/usr/logo.png
OpenAI | https://openai.com/ | AI 产品与研究 |

# 博客
示例博客 | https://example.com/ | 这是一个示例站点 |
<!-- timellow-links-end -->
```

每一条友情链接的格式为：

```text
名称 | URL | 描述 | 图标URL
```

说明：

- `名称` 和 `URL` 必填
- `描述` 可选
- `图标URL` 可选
- 以 `# 分组名` 开头表示新的分组

### 不写注释块也可以

如果整个友情链接页面内容本身就是友链数据，主题也会直接解析整页内容。

示例：

```text
# 设计
Typecho | https://typecho.org/ | 轻量开源博客程序 |
OpenAI | https://openai.com/ | AI 产品与研究 |
```

这种写法下，页面内容会被当作纯友链数据处理，不再额外显示介绍区域。

## 独立页面

以下 `slug` 会自动匹配到对应模板：

- `links`
- `archives`
- `categories`
- `tags`

## 文章列表加载方式

主题支持两种文章列表切换方式：

- `页码`
- `加载更多`

当选择 `加载更多` 时，首页和归档类列表会使用接近 `index.html` 的按钮样式逐页追加文章；如果浏览器脚本不可用，按钮仍然会回退为普通下一页链接。

## 说明

- 默认封面优先级：
  1. 文章自定义字段 `cover`
  2. 正文第一张图片
  3. 主题设置中的 `默认缩略图`
  4. `assets/cover` 目录中的随机封面
- 友情链接页、评论区和文章正文中的长链接会自动换行
- 代码块带语言标题和复制按钮
